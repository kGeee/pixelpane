## Known Architectural Decisions

- App is menu-bar-first using `MenuBarExtra`.
- App is configured as `LSUIElement`.
- App sandbox is disabled to align with Direct distribution.
- Pixel Pane alpha and v1 target macOS 15.2+ because `SCScreenshotManager.captureImage(in:)` requires it.
- Alpha global hotkey uses Carbon `RegisterEventHotKey`; `CGEventTap` is deferred unless Carbon fails QA.
- **Local-First AI Default (2026-04-29):** every AI action defaults to on-device. Apple Foundation Models handles text-only local AI when available; optional MLX/VLM setup handles local image understanding. Cloud Mode is an explicit Settings opt-in, and the current single Cloud Mode toggle covers cloud-capable text and image context.
- **Local Vision Runtime Via MLX (2026-04-29):** image-aware Local Mode should use an installed MLX vision model. Current recommended setup model is `mlx-community/Qwen3.6-35B-A3B-6bit` when compatible with the user's Mac.
- **Cloud Proxy Contract And Hosting (2026-04-29):** the first cloud-upgrade backend is a Cloudflare Workers proxy in front of Anthropic. The macOS app talks to the Pixel Pane `/v1` contract in `docs/backend-api.md` and never stores provider API keys.
- **Sparkle Release Update Process (2026-05-21):** Direct distribution updates use Sparkle with a planned production appcast at `https://pixelpane.app/appcast.xml`; beta builds may temporarily use an HTTPS release-site appcast. Sparkle EdDSA private keys are release secrets and must not be committed, embedded in the app, or deployed to the backend.
- **Notch-Native Local Assistant (2026-05-21):** Pixel Pane's core surface is now a hover-open notch assistant. Plain chat can open without capture context; captures create a fresh contextual Ask session; local remains default and Cloud Mode is a single explicit routing choice.
- **Chat-Only Notch Surface (2026-05-21):** The visible notch UI is now a single chat assistant. Extract, Translate, Explain, and Simplify are natural-language capabilities through chat/context, not top-level tabs.
- **User-Granted Read-Only Local Files (2026-05-21):** The notch assistant may read/search only files or folders explicitly granted by the user. File snippets stay local in Local Mode and may be sent only when the user routes the chat through Cloud Mode. Create/edit/delete/move tools are deferred to a separate confirmed-write story.
- **Local Chat History Without Screenshot Retention (2026-05-21):** Chat transcripts are stored locally so users can explicitly reopen saved conversations. Screenshots and attached-image pixels are not retained in chat history; as of 2026-05-24, bounded source/snippet/OCR metadata may be stored in assistant tool state for follow-up continuity.
- **Local Text Runtime Via MLX (2026-05-21):** Local Mode can use a selected text-only MLX model for text chat/actions through `mlx_lm.generate` when setup passes. MLX Vision remains separately gated on a vision-capable model and `mlx_vlm.generate`.
- **Confirmed Local File Writes (2026-05-21):** The assistant may stage local file creation or text edits only inside user-granted file/folder locations. Writes require a visible confirmation naming the target path before any file is changed; model output never directly mutates files.
- **App-Owned Model-Agnostic Assistant Harness (2026-05-24):** The assistant now has an app-owned capability contract and Ask preflight router for deterministic answers, local file grant answers, local write proposals, and file-search gating. Native model tool calling remains optional; file grants and side-effect policy are enforced by Pixel Pane.
- **Transient User Image Context (2026-05-24):** Users can attach an image to the active assistant chat. The image stays in memory, is OCR'd locally for text-only fallback, may route to MLX Vision or Cloud Mode when allowed, and is not persisted into chat history.
- **Persistent Assistant Tool State And Context Packing (2026-05-24):** Each chat can persist bounded assistant tool state: source metadata, last listed folder, recent file snippets, visual/OCR context metadata, and recent tool summaries. Context sent to models is packed by source group, marks retrieved file/OCR/image/tool text as untrusted data, and adapts to route capability budgets instead of depending on native model tool calling.
- **Bounded Local Terminal Tool (2026-05-24):** The assistant may run terminal commands only through Pixel Pane's app-owned tool layer. Commands are validated, bounded by timeout/output caps, recorded as tool results, and risky/destructive commands require visible confirmation; disallowed shell-bomb patterns are blocked.
- **Terminal-Backed File Discovery Before Model Chat (2026-05-24):** Specific file existence/search questions now use deterministic terminal-backed discovery inside user-granted folders before model chat or generic grant inventory. Found paths are recorded as file sources for follow-up reads and confirmed edit proposals.
- **Mode-Independent Agentic Tool Loop (2026-05-25):** Pixel Pane's app harness now runs safe file discovery/read steps before either Local or Cloud model generation. Local Mode keeps terminal/file/model work local; Cloud Mode receives only explicitly packed local context after the same app-owned tools have run.
- **General Terminal Agent With Risk-Based Approval (2026-05-25):** The app-owned harness can plan and run broad terminal commands, not only file/repo commands. Low-risk read-only system inspection can run automatically; write-like, build/script, network, install, process-control, privileged, and system-affecting commands require visible confirmation. Local Mode remains fully local.
- **Selected-Model Planning For Delegated Writes (2026-05-26):** Natural write requests that delegate filename/content choices to the assistant are planned by the selected local/cloud model, then converted into app-owned confirmed write proposals. Pixel Pane still validates grants and requires visible confirmation before writing.
- **Current-Session Write Grounding And Fresh Chats (2026-05-26):** Delegated write planning may use bounded prior turns from the active chat to resolve references such as "these results," but fresh assistant chats start with empty turns/tool state and do not auto-load the latest saved chat. Saved chats are reintroduced only by explicit user selection from history.
- **Model-Agnostic Recent-File Rewrite Flow (2026-05-26):** Rewrite/format follow-ups for a recently staged/read file are handled by the app-owned harness: Pixel Pane resolves and reads the granted file, then asks the selected model only to produce replacement content for a confirmation-gated write.
- **App-Owned Named-Grant Targeting For Writes (2026-05-26):** When the user names a granted folder for a delegated write, Pixel Pane resolves the intended grant itself, including bounded typo matching, and constrains selected-model targets to that grant before confirmation.
- **App-Owned Named-Grant Targeting For Folder Listings (2026-05-26):** Folder content questions now use the same app-resolved preferred granted folder as writes, so exact names and bounded typos such as `pixel=pane-tests` list `pixel-pane-test` instead of falling through to model chat or a stale broader grant.
- **App-Owned Folder-Listing File Follow-Ups (2026-05-26):** Folder listings now record visible child files as recent file sources, so follow-ups such as `that txt file`, `read it`, or `sure do it` can trigger a real Local Files read before the selected model is asked to answer.
- **State-First Recent Source References (2026-05-26):** Deictic source questions such as `what are these files?` are answered from the latest structured tool observation before broad grant inventory, file search, or model context generation.
