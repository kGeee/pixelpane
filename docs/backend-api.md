# Pixel Pane Backend Proxy API Contract

Status: accepted for implementation.

This contract defines the first cloud-upgrade API for Pixel Pane. Local Mode remains the default. The macOS app may call this proxy only when Cloud Mode is enabled and the user has granted any required per-action image consent.

## Hosting And Ownership

- Runtime: Cloudflare Workers.
- Provider: Anthropic Messages API through a server-side API key.
- Client secrets: the macOS app never stores Anthropic, Stripe, RevenueCat, or Worker secrets.
- Streaming: the proxy returns `text/event-stream` responses using Server-Sent Events.
- Logging default: prompt content, screenshots, OCR text, user questions, glossary matches, and model output are not logged by default.

## Base URL And Versioning

Production base URL:

```text
https://api.pixelpane.app/v1
```

Development base URL:

```text
https://dev-api.pixelpane.app/v1
```

Every action request must include:

```http
Authorization: Bearer <pixel-pane-app-token>
Content-Type: application/json
Accept: text/event-stream
X-PixelPane-Client-Version: <bundle-short-version>
X-PixelPane-Request-ID: <uuid>
```

`X-PixelPane-Request-ID` is generated by the app per action attempt and is safe to log. It must never encode OCR text, prompt text, filenames, or user-visible content.

## Endpoints

All action endpoints use `POST` and stream a response.

| Endpoint | Purpose | Supports image payload |
|---|---|---|
| `/translate` | Translate captured text to the user target language | No |
| `/explain` | Explain captured text or screenshot context | Yes, only with per-action image consent |
| `/simplify` | Rewrite captured text in simpler language | No |
| `/ask` | Answer a follow-up question about a capture | First turn only, only with per-action image consent |
| `/study` | Study-mode explanation with terms and learning help | Yes, only with per-action image consent |
| `/menu` | Menu translation with cultural context | Yes, only with per-action image consent |
| `/debug` | Technical/code/log/error help | Yes, only with per-action image consent |

### Auth Token Endpoint

`POST /auth/token` issues a short-lived Pixel Pane app token for an anonymous device ID. It does not require an existing bearer token.

Request:

```json
{
  "schema_version": "2026-04-29",
  "device_id": "a random UUID generated once by the app",
  "client_context": {
    "app_version": "0.1.0",
    "platform": "macOS",
    "locale": "en-US",
    "timezone": "America/Los_Angeles"
  }
}
```

Response:

```json
{
  "schema_version": "2026-04-29",
  "token": "<pixel-pane-app-token>",
  "token_type": "Bearer",
  "expires_at": "2026-04-30T17:30:00.000Z",
  "device_id": "the same anonymous device ID"
}
```

The device ID is stored in Keychain by the app and is used for anonymous free-tier quota. Sign in with Apple remains deferred until upgrade/account work.

## Shared Request Schema

All endpoints accept this envelope. Fields that do not apply to an action are omitted, not sent as `null`.

```json
{
  "schema_version": "2026-04-29",
  "action": "translate",
  "capture": {
    "source_type": "ocr",
    "ocr_text": "Bonjour",
    "detected_language": "fr",
    "created_at": "2026-04-29T17:30:00Z"
  },
  "target_language": "en",
  "question": "What does this warning mean?",
  "conversation": [
    {
      "role": "user",
      "content": "What does this error mean?"
    },
    {
      "role": "assistant",
      "content": "It means..."
    }
  ],
  "image": {
    "mime_type": "image/png",
    "data_base64": "<base64-png>",
    "user_consented": true
  },
  "client_context": {
    "app_version": "0.1.0",
    "platform": "macOS",
    "locale": "en-US",
    "timezone": "America/Los_Angeles",
    "cloud_mode": true
  },
  "limits": {
    "max_output_tokens": 768
  }
}
```

### Field Rules

- `schema_version` is required and must be the date string for this contract until a later version is documented.
- `action` is required and must match the endpoint: `translate`, `explain`, `simplify`, `ask`, `study`, `menu`, or `debug`.
- `capture.source_type` is required. Initial value: `ocr`.
- `capture.ocr_text` is required for all endpoints. Empty or whitespace-only text returns `400 empty_ocr_text`.
- `capture.detected_language` uses a BCP 47 language code when known. Omit when unknown.
- `target_language` is required for `/translate` and optional elsewhere.
- `question` is required for `/ask`.
- `conversation` is allowed only for `/ask`; the app sends at most five turns and must not resend image data after the first turn.
- `image` is allowed only for image-capable endpoints. The proxy rejects image payloads unless `image.user_consented` is `true`.
- `image.data_base64` should be PNG or JPEG, capped at 5 MB after base64 decoding.
- `client_context` is metadata only. It must not include OCR text, prompt text, result text, filenames, or window/app names captured from the user's screen.
- `limits.max_output_tokens` is optional. The server clamps it to the plan and action maximum.

## Streaming Response Format

The proxy normalizes provider streaming into Pixel Pane events instead of forwarding raw Anthropic events.

Response headers:

```http
HTTP/1.1 200 OK
Content-Type: text/event-stream; charset=utf-8
Cache-Control: no-store
X-PixelPane-Request-ID: <uuid>
```

Event sequence:

```text
event: meta
data: {"request_id":"...","action":"explain","model":"server-configured","plan":"free","remaining_cloud_actions":9,"reset_at":"2026-04-30T00:00:00Z"}

event: snapshot
data: {"text":"The first partial answer"}

event: snapshot
data: {"text":"The first partial answer with more text"}

event: done
data: {"stop_reason":"end_turn","input_tokens":421,"output_tokens":116}
```

Event semantics:

- `meta`: emitted first. Contains non-sensitive request metadata and quota state.
- `snapshot`: emitted zero or more times. The app replaces the active result text with `text`; these are snapshots, not deltas.
- `note`: optional inline status, such as `{"code":"cloud_fallback","message":"Cloud unreachable, used local"}` once local fallback exists.
- `error`: terminal stream error with the same schema as non-streaming errors.
- `done`: terminal success event. The app clears loading state after this event.

The app must ignore unknown event names for forward compatibility.

## Error Responses

Non-streaming failures use JSON:

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Cloud action limit reached.",
    "retry_after_seconds": 3600,
    "request_id": "..."
  }
}
```

Streaming failures use:

```text
event: error
data: {"error":{"code":"provider_overloaded","message":"Cloud model overloaded.","retry_after_seconds":30,"request_id":"..."}}
```

Error codes:

| HTTP | Code | App behavior |
|---|---|---|
| 400 | `invalid_request` | Show generic action error; do not retry automatically |
| 400 | `empty_ocr_text` | Show no-text empty state |
| 400 | `image_not_allowed` | Retry without image or ask for consent |
| 401 | `unauthorized` | Refresh app token; if refresh fails, show account recovery |
| 403 | `cloud_mode_disabled` | Run locally instead |
| 413 | `payload_too_large` | Retry without image or ask user to capture a smaller region |
| 429 | `rate_limited` | Show upgrade path and Run locally instead |
| 500 | `server_error` | Run locally instead |
| 502 | `provider_error` | Run locally instead |
| 529 | `provider_overloaded` | Run locally instead |

## Rate Limits And Quota

- Free tier target: 10 cloud actions per UTC day.
- Enforced server-side by anonymous device/account token.
- `meta.remaining_cloud_actions` and `meta.reset_at` let the app show quota without logging content.
- Rate limit response includes `retry_after_seconds` when available.
- Local OCR and local AI actions are not rate limited by the proxy.

## Auth Contract

Initial implementation uses a bearer app token:

```http
Authorization: Bearer <pixel-pane-app-token>
```

The token is issued by the future anonymous device identity/auth flow and scoped to:

- device or user account ID
- plan tier
- token expiry
- allowed API version

The proxy must not accept provider API keys from the app. All provider keys live only in backend secrets.

## Retention And Logging Policy

Default logs may include:

- timestamp
- endpoint/action
- HTTP status/error code
- request ID
- app version
- token/account hash
- plan tier
- latency
- input/output token counts
- payload byte size

Default logs must not include:

- OCR text
- screenshots or image base64
- prompt text
- user questions
- model output/result text
- glossary entries
- filenames, window titles, or source application names

Debug logging that includes user content is not allowed in production. Any future opt-in diagnostic bundle must require explicit user action and must be documented separately.
